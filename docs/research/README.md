# Research

Prior art, source-level peer studies, and corpus/real-data findings that justify the design.

- [`prior-art.md`](prior-art.md) — battle-tested systems we mine for ideas (Lucene Monitor/Luwak, ES/OpenSearch percolator, Tantivy/Quickwit, roaring bitmaps, Aho-Corasick/daachorse, set-containment joins) + a synthesis table.
- [`broad-scaling-prior-art.md`](broad-scaling-prior-art.md) — the complement: what the systems that are excellent at boolean-expression matching at scale (Yahoo's k-index→Vespa lineage, BE-Tree/PS-Tree/A-Tree, SIFT/Le Subscribe/NiagaraCQ/YFilter, Siena covering, LEO/CE-feedback self-tuning) do about **hot/broad predicates, duplicate queries, and statistics-driven classification** — triggered by ADR-104's measured broad-lane growth; ends in five evidence-ranked levers + their FN-safety arguments (a future roadmap program, pending review).
- [`percolator-workload.md`](percolator-workload.md) — how a production percolator is *operated* (abstract reference workload): per-query metadata tags, filter-by-tag as the dominant read pattern, the two-stage recall+verify, and where ranking is/isn't used — plus how Reverse Rusty maps to each (→ [`../DECISIONS.md`](../DECISIONS.md) ADR-049, [`../STATUS.md`](../STATUS.md) Tier 4).
- [`clustering-prior-art.md`](clustering-prior-art.md) — consistent-hashing variant comparison (ring+vnodes chosen), content-based pub/sub routing, and the Elasticsearch distributed-percolator contrast that ground the clustering design (→ [`../DECISIONS.md`](../DECISIONS.md) ADR-027).
- [`dynamic-vocabulary.md`](dynamic-vocabulary.md) — **charter** for the Cluster v1 spike on absorbing new vocabulary after the dict is frozen: ES/OS global ordinals + dynamic mapping, Vespa real-time indexing, RocksDB shared-dictionary versioning, Lucene FST, and deterministic feature-hashing — evaluated against the zero-false-negative contract (→ ADR-046).
- [`corpus-feature-learning.md`](corpus-feature-learning.md) — learning the feature extractor from the query corpus (NPMI entity induction), what's safe to learn vs the aliasing safety rail, measured learner results.
- [`real-data-findings.md`](real-data-findings.md) — testing the normalizer against real eBay "PSA 10" titles: bugs found & fixed, and the aspects-first + corpus-learned front-end conclusion.
- [`multiword-synonyms.md`](multiword-synonyms.md) — **design learnings** from an abandoned first attempt at ES-`synonym_graph`-equivalent multi-word aliases: single-token is easy (equivalence expansion); multi-word is a *matching-model* (token-graph) feature whose flat-feature-set approximation fundamentally conflicts with forbidden-feature checks. Records the approaches tried, the two walls, and the recommended two-feature-set direction for the next attempt.
